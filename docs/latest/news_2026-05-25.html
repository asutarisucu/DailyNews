🛒 Shopify 開発者情報

Shopify CLI 4.0 リリース — セマンティックバージョニング・自動アップデート・非推奨フラグ完全削除

shopify
🔗 Shopify Developer Changelog

Shopify CLI がメジャーバージョン 4.0 に到達し、セマンティックバージョニング（MAJOR.MINOR.PATCH）を正式採用した。4.0 からはインストールに使用したパッケージマネージャーで自動アップグレードが行われ、CI 環境・プロジェクトローカルインストール・メジャーバージョン更新は除外される（shopify config autoupgrade off で無効化可）。昨日までに非推奨を告知していた app deploy / app release の --force フラグ、および shopify webhook trigger（旧コマンド）がこのバージョンで完全に削除された。影響を受けるスクリプトや CI/CD パイプラインは --allow-updates / --allow-deletes フラグを使った粒度の細かい制御へ移行が必要。

Shop Minis 最新アップデート — Optional Consent がデフォルト化・3 つの新フック・Intent による文脈起動に対応

shopify
🔗 Shopify Developer Changelog

Shop Minis の最新アップデートが公開され、Optional Consent（任意同意）がデフォルトの挙動となった。useConsentStatus・useGrantConsent・useDenyConsent の 3 つの新フックが追加され、スコープが拒否された場合でもミニアプリが機能を縮小しながら動作し続けられるようグレースフルデグレードを実現できる。また Intents が新たなコミュニケーションレイヤーとして導入され、try_on と view_in のインテントにより Shop アプリ内の文脈に合わせたタイミングでミニアプリを起動できる。pnpm サポート・非同期ストレージ制限の見直し・availableForSale による在庫切れハンドリングも追加されている。

Shopify マーチャント向けに、ディスカウントを特定マーケットに限定して割り当てる機能が一般提供（GA）となった。Discount Details ページの eligibility セレクターで任意のコードベース・自動ディスカウントを 1 つ以上のマーケットに関連付けられるほか、ディスカウント一覧でのマーケット・顧客フィルタリング、Market Details ページでそのマーケットに割り当てられた全ディスカウントの一覧表示が可能になった。地域限定セールや B2B 顧客向けの特別価格をきめ細かく制御できるようになり、マーチャントにとって大幅な柔軟性向上となる。

Shopify Scripts 廃止まで 36 日 — 6/30 の最終期限が迫る、Shopify Functions への移行を急いで

shopify
🔗 Shopify Developer Changelog

Shopify Scripts の廃止期限（2026 年 6 月 30 日）まで残り 36 日となった。すでに 4 月 15 日を過ぎているため新しい Script の編集・公開は不可能で、6/30 以降はすべてのカスタムディスカウント・送料・支払い Script の実行が完全に停止する。置き換えとなる Shopify Functions（WebAssembly ベース）との共存期間は終了が近く、未移行のストアでは割引・送料ロジックが突然機能しなくなるリスクがある。Shopify Scripts customizations report で移行対象を特定し、Shopify Functions または公開アプリへの代替を速やかに完了する必要がある。

CVE-2026-42897 — Microsoft Exchange OWA ゼロデイ XSS、細工メール 1 通で任意 JS 実行・能動的悪用が確認済み

security
🔗 The Hacker News

Microsoft が CVE-2026-42897（CVSS 8.1）のセキュリティアドバイザリを公開し、Outlook Web Access（OWA）に存在するクロスサイトスクリプティング（XSS）の脆弱性が既に能動的に悪用されていることを確認した。攻撃者が細工したメールをユーザーに送信し、そのユーザーが OWA で開くだけでブラウザコンテキストで任意の JavaScript が実行される。Exchange Server 2016・2019・SE RTM のオンプレミス版が対象（Exchange Online は影響なし）。EM Service 有効の環境には自動緩和策が配信済みで、EOMT スクリプトでも適用可能。永続的パッチは現在開発中。
