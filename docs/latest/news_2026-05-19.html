Checkout & Accounts Configuration API (2026-04) リリース — 旧 Checkout Profile API・Checkout Branding API を統合・廃止、Plus 限定で統一ブランディングを一元管理

shopify
🔗 shopify.dev/changelog

API バージョン 2026-04 で新たに提供開始された Checkout And Accounts Configuration API は、チェックアウト・カスタマーアカウント・サインインページのブランディングを単一 API で統一管理できる。共有 designTokens・コンポーネントを使うことでブランド設定を一度行うだけで全サーフェスに反映される。Checkout Profile API と Checkout Branding API は両 API ともに非推奨となり、Shopify Plus マーチャント専用の機能として提供される。

Storefront API 2026-07 — カート内地域非対応商品に PRODUCT_UNAVAILABLE_IN_BUYER_LOCATION 警告を自動付与、フロントエンドでのユーザー通知が容易に

shopify
🔗 shopify.dev/changelog

Storefront API バージョン 2026-07 より、バイヤーの所在地で購入できない商品がカートに含まれる場合、Cart オブジェクトが PRODUCT_UNAVAILABLE_IN_BUYER_LOCATION 警告を返すようになった。対象カートライン1件ごとに個別の警告が付与され、warning.target にカートライン ID が入るため UI での精度の高いエラー表示が可能になる。越境 EC や B2B ロケーション制限を実装しているヘッドレスストアは、2026-07 へのバージョンアップ時に対応が必要。

全アプリで CI/CD トークンデプロイが解禁 — Dev Dashboard 発行の App Automation Token で GitHub Actions 等から完全自動デプロイ、旧 Partner Dashboard トークンを廃止

shopify
🔗 Shopify Developer Changelog

これまで一部アプリに限定されていた非インタラクティブ環境向けデプロイが全アプリで利用可能になった。新しい App Automation Token は Dev Dashboard で発行し、環境変数 SHOPIFY_APP_AUTOMATION_TOKEN にセットするだけで Shopify CLI の deploy コマンドが動作する。トークンはアプリ単位でスコープが限定され、有効期限（1・3・6 か月）とローテーションワークフローも Dev Dashboard から管理できる。旧 SHOPIFY_CLI_PARTNERS_TOKEN 方式からの移行が推奨される。

Claude Managed Agents に3新機能 — dreaming（過去セッション学習・自律的自己改善）、multiagent orchestration（並列スペシャリスト委任）、共有ファイルシステムを追加

Anthropic は Claude Managed Agents に3つの新機能を追加した。「dreaming」（リサーチプレビュー）は過去セッションを振り返りパターンを学習してエージェントが自己改善を行う長期記憶機能。「multiagent orchestration」はリードエージェントがタスクを分解し、それぞれ固有のモデル・プロンプト・ツールを持つスペシャリストエージェントに並列委任する仕組み。加えて共有ファイルシステムにより複数エージェント間でのデータ受け渡しが簡素化された。

🔗 InfoQ

【CISA KEV】CVE-2026-31431「Copy Fail」— Linux カーネル AF_ALG サブシステムの LPE、2017年以降の全主要ディストリに影響、732 バイトの Python スクリプトで確実に root 奪取

security
🔗 The Hacker News

CISA が CVE-2026-31431（CVSS 7.8）を Known Exploited Vulnerabilities カタログに追加し、FCEB 機関に 5/15 までのパッチ適用を義務付けた。カーネルの AF_ALG（ユーザースペース暗号 API）の algif_aead モジュムに存在するロジック欠陥で、splice() システムコールを悪用することでページキャッシュへの4バイト書き込みが可能となり、/usr/bin/su 等の特権バイナリをメモリ上で改竄して root 権限が取得できる（ディスク上ファイルは無変更）。Ubuntu・Amazon Linux 2023・RHEL 10.1・SUSE 16 等で再現確認済み。修正済みバージョンは Linux 6.18.22・6.19.12・7.0。

Deno Deploy Classic（dash.deno.com）が 2026/7/20 に廃止 — 新規プロジェクトは console.deno.com へ移行必須、既存プロジェクトも移行対応が必要

dev
🔗 Deno Blog

Deno Deploy の旧プラットフォーム（dash.deno.com）が 2026年7月20日をもってサービス終了する。新規プロジェクトはすでに更新版プラットフォーム（console.deno.com）での作成が推奨されており、既存プロジェクトも移行期限までに対応が必要。移行作業は Deno 公式ドキュメントのガイドを参照。今回の Deno 2.6 リリースと並行したタイミングでのアナウンスとなった。

🔗 GitHub Changelog

GitHub Copilot App（テクニカルプレビュー）が公開された。GitHub ネイティブのデスクトップ体験として、進行中の作業を起点にエージェント開発を開始し、プルリクエストのレビューを経て変更をランディングできるワークフローを提供する。同時に個人プランの変更も発表され、既存ユーザー体験を守るため新規 Pro 登録を一時停止。Opus モデルは Pro プランから除外され Pro+ プランのみで利用可能となった。VS Code・Copilot CLI では使用上限接近時に残りトークン量が可視化されるようになった。
