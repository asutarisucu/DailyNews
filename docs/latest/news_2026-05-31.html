🛒 Shopify 開発者情報

1注文でシップ＆ピックアップを同時サポート — フィーチャープレビュー公開、配送と店舗受け取りの混在フルフィルメントが可能に

shopify
🔗 Shopify Developer Changelog

Shopify Plus / Enterprise ストアで、シップと店舗ピックアップを1回の注文に混在させられる機能がフィーチャープレビューとして公開された。API に破壊的スキーマ変更はないが、Order が複数の fulfillmentOrder を持てるようになり、各 fulfillmentOrder が異なる delivery_method（SHIPPING または PICK_UP）を保持できる。配送・フルフィルメント情報の読み取り・計算・表示を行うアプリは動作への影響が生じる可能性があるため、Plus プランのデベロッパーストアでフィーチャープレビューを有効化し、早急なテストが推奨される。

管理画面 Settings インテント 7種を追加 — 通知・決済キャプチャ・ギフトカード・配送プロファイル・事業者詳細に API 1回で直接遷移

shopify
🔗 Shopify Developer Changelog

3月に導入した Settings インテントが7種類拡張された。通知・決済キャプチャ・ギフトカード・配送プロファイル・事業者詳細の各設定画面に対応し、アプリから API を1回呼び出すだけで Shopify 管理画面の該当 Settings ページがコンテキストオーバーレイとして開き、対象フィールドへ自動スクロールされる。マーチャントに複雑な操作を強いることなく必要な設定へ誘導するワークフローの実装が容易になった。

shopify
🔗 Shopify Developer Changelog

API バージョン 2026-04 から、Checkout Profile API と Checkout Branding API が廃止され、新しい Checkout And Accounts Configuration API に統合された（Shopify Plus 限定）。designTokens とコンポーネントでブランディングを一度設定すれば、チェックアウト・カスタマーアカウント・サインイン全ページへ一貫したスタイルが自動適用される。表面固有のロゴ・カラー・セクションスタイルの上書きも可能。Checkout Branding API を使用しているアプリは移行対応が必要。

🔒 セキュリティ

Verizon DBIR 2026 — 脆弱性悪用が資格情報窃取を抜いて19年来初のトップ侵入経路に、サードパーティ侵害が前年比60%増の48%へ急増

security
🔗 Verizon DBIR 2026

Verizon の2026年データ侵害調査レポート（DBIR）が発表された。ソフトウェア脆弱性の悪用が侵入経路の31%（前年20%から上昇）を占め、レポート19年の歴史で初めて資格情報窃取をトップから引き下ろした。ランサムウェアは侵害の48%に登場し、サードパーティ経由の侵害は前年比60%増の48%に拡大。CVE 修復の中央値は43日（前年32日）に悪化し、CISA KEV カタログ内の重大 CVE を完全修復した組織は38%から26%に減少した。Shadow AI の利用が前年比3倍の45%に上昇したことも報告されている。

LLMエージェントが自律的に侵害を拡大する攻撃を初観測 — Marimo CVE-2026-39987 RCE 悪用後、AWS クレデンシャル窃取〜内部DB流出まで約1時間で完遂

security
🔗 The Hacker News

Sysdig が、Marimo ノートブックの事前認証 RCE 脆弱性 CVE-2026-39987（Marimo 0.20.4以前）を悪用後に LLMエージェントを使い自律的に侵害を拡大する攻撃を初観測した（5月10日）。エージェントは AWS IMDS・Secrets Manager からクレデンシャルを抽出し、SSH キーで内部バスチョンを4経路で経由して PostgreSQL データベースのスキーマと全データを約2分で窃取。HEREDOC でコンテキストウィンドウを管理しながら出力をまとめる LLMエージェント特有の痕跡を確認した。Marimo 0.23.0 以降で修正済みのため早急なアップデートと全クレデンシャルのローテーションが必要。

PAN-OS CVE-2026-0257（GlobalProtect 認証バイパス）が積極悪用中 — CISA が KEV 追加、Rapid7 が5月17日以降2波の攻撃キャンペーンを観測

security
🔗 The Hacker News

Palo Alto Networks PAN-OS の GlobalProtect に存在する認証バイパス脆弱性 CVE-2026-0257（CVSS 7.8）が野生で積極的に悪用されており、CISA が5月29日に KEV カタログへ追加した。「認証オーバーライド」機能のセッションクッキーが HTTPS サービスと同一証明書を共有する場合にのみ発現し、攻撃者は正規エンドポイントを偽装して未認証で VPN 接続を確立できる。Rapid7 は5月17日の第1波（Vultr 経由）と5月21日の第2波（Dromatics Systems 経由）を確認しており、一部の被害者では VPN IP アドレスが付与され内部ネットワークへの直接アクセスが可能になった。5月13日公開の修正パッチへの早急な適用が必要。

CISA KEV 5月27日更新 — LiteSpeed cPanel 特権昇格・BerriAI LiteLLM SQL インジェクション・Daemon Tools マルウェアの3件を追加

security
🔗 CISA

CISA が5月27日に Known Exploited Vulnerabilities（KEV）カタログへ3件の脆弱性を追加した。LiteSpeed cPanel プラグインの特権昇格（任意の cPanel ユーザーがルート権限でスクリプトを実行可能）、BerriAI LiteLLM の SQL インジェクション（プロキシDB からの認証なしデータ読み取り・改ざん）、Daemon Tools Lite の埋め込みマルウェアコードが対象。連邦機関には期限内の修復が義務付けられており、民間組織にも迅速な対応が推奨される。特に LiteLLM は LLM プロキシとして広く使用されているため影響範囲が大きい。

cloud

Microsoft Defender for Open-Source Relational Databases が2026年6月1日より AWS RDS インスタンスに対して一般提供（GA）となり、同日から課金も開始する。これにより Azure 以外のクラウド環境の RDS インスタンスに対しても脅威検出・アラート・セキュリティ推奨事項が提供される。マルチクラウド戦略を採用する組織が Microsoft Defender for Cloud の一元管理ダッシュボードで AWS の DB リソースもカバーできるようになる。対象は PostgreSQL・MySQL・MariaDB などのオープンソース RDB。

Vercel CLI に Node.js 不要のネイティブバイナリが実験的提供開始 — コード署名済み・macOS Keychain 連携、CI/コンテナ環境でのセットアップを大幅簡略化

dev
🔗 Vercel Changelog

Vercel CLI（5月27日）に、Node.js ランタイム不要で動作するネイティブバイナリがオプトインの実験的機能として追加された。バイナリはコード署名済みで OS が Vercel の正規バイナリを検証でき、macOS ではシステム Keychain にスコープされた認証情報を保存するため他のプロセスからアクセスできない。macOS・Linux・Windows（x64/arm64）に対応し、pnpm i -g @vercel/vc-native -f でオプトイン可能。Node.js の管理が不要な軽量コンテナ・CI ジョブ・マネージド開発ワークスペースでのメリットが大きい。

悪意ある npm パッケージ 33件が Dependency Confusion で開発者環境をプロファイリング — 9組織スコープを偽装し C2 から難読化ペイロードを取得

dev
🔗 Microsoft Security Blog

Microsoft Security が5月29日、Dependency Confusion 手法を悪用した33件の悪意ある npm パッケージを公開した。3つのメンテナーエイリアスで2回に分けて公開（5月28-29日）され、9つの組織スコープの内部パッケージを偽装。package.json に GitHub Enterprise・Jira・ドキュメントポータルの社内 URL を記載して正規パッケージに見せかけ、インストール後に難読化された postinstall スクリプトが C2 サーバーからペイロードをダウンロード・実行する。5月28日以降に該当バージョンをインストールした環境はクレデンシャルの全ローテーションが必要。npm install --ignore-scripts の使用が緊急推奨される。

pnpm 11 が Minimum Release Age をデフォルト有効化 — npm サプライチェーン攻撃への対策として新規パッケージの即時インストールを72時間ブロック

dev
🔗 CyberSecurity News

pnpm 11 が「Minimum Release Age」（最低リリース経過時間）機能をデフォルトで有効化した。新規公開されたパッケージバージョンは公開から一定時間（デフォルト72時間）が経過するまでインストールできなくなる。これにより、TanStack や Mistral AI などで相次いだ npm サプライチェーン攻撃のように直後に悪意ある内容に置き換えられたバージョンへの即時感染を抑止できる。攻撃者が汚染バージョンを即時インストールさせる時間的余裕を奪うことで、セキュリティチームやパッケージメンテナーが検知・対応する猶予を生み出す。
