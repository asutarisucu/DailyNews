shopify
🔗 Shopify Developer Changelog

Liquid ストアフロントに標準の通信レイヤーが追加され、テーマが shopify:product:view・shopify:cart:lines-update などの DOM イベントを発行し、アプリやエージェントが JavaScript でサブスクライブして追加の API 呼び出しなしでペイロードを受け取れるようになった。Shopify.actions.update などのアクションが全 Liquid ストアフロントで利用可能になり、アプリやエージェントがテーマの動作をトリガーできる。テーマ開発者はアクションをオーバーライドしてページリロードなしの UI 更新を実装できる。この標準化により、テーマ間での一貫したアプリ・エージェント連携が可能になり、Shopify のエージェント対応ストアフロント戦略の中核となる。

npm v12 破壊的変更予告: インストールスクリプトが7月よりデフォルト無効化 — CI/CD パイプラインの事前対応が必要

dev
🔗 GitHub Changelog

npm v12（7月リリース予定）では allowScripts がデフォルト off となり、依存パッケージの preinstall・install・postinstall スクリプトが明示許可なしでは実行されなくなる。--allow-git もデフォルト none に変更されて Git 依存関係の解決が制限され、--allow-remote も none となりリモート URL からの依存関係解決が不可になる。node-gyp ネイティブビルドも影響を受けるため、npm approve-scripts コマンドで許可リストを整備して package.json にコミットしておく必要がある。v11.16.0 以降では今すぐ警告付きで動作確認が可能で、サプライチェーン攻撃の代表的な手口「Phantom Gyp」への直接的な対策となる。
