Storefront API 2026-07: Cart に配達オプション利用不可の警告フィールドを追加 — サイレント変更を防止

shopify
🔗 shopify.dev/changelog

GraphQL Storefront API バージョン 2026-07 から、バイヤーが選択した配達オプションが利用不可になった場合に Cart オブジェクトの warnings フィールドへ警告が追加されます。これまでは住所変更やカート内容の変更で配達方法が自動的に切り替わっても通知されませんでしたが、今後は明示的な警告コードで状態を把握できます。また API バージョン 2024-10 以降では在庫エラーも userErrors ではなく warnings フィールドで返されるため、カートミューテーションのエラーハンドリングの見直しが推奨されます。

ProductVariant が Publishable に — チャネルごとにバリアント単位で公開状態を制御可能に

shopify
🔗 shopify.dev/changelog

ProductVariant オブジェクトが Publishable インターフェースを実装したことで、チャネル単位でバリアントの公開・非公開を細かくコントロールできるようになりました。これまで商品全体でしか公開制御できなかったところ、特定バリアント（サイズ・カラーなど）をオンラインストアには表示しつつ卸売チャネルには非表示にする、といった柔軟な在庫・販売戦略が GraphQL Admin API から実現できます。マルチチャネル展開アプリにとって重要なアップデートです。

security
🔗 The Hacker News

CISA が Linux カーネルのローカル特権昇格脆弱性（CVE-2026-31431, CVSS 7.8）を既知悪用脆弱性カタログ（KEV）に追加しました。非特権のローカルユーザーが root アクセスを取得できる脆弱性で、実際の悪用が確認されています。FCEB 機関への修正期限は 5/15 に設定されており、Linux ベースのサーバー・コンテナ基盤を運用する組織は OSカーネルのバージョン確認とパッチ適用を早急に行ってください。クラウド VM・オンプレサーバーいずれも対象です。

security
🔗 CISA

CISA が 5月4日週分の脆弱性サマリー（SB26-131）を公開しました。CVSS 7.0 以上の高・緊急レベル脆弱性が多数掲載されており、OS・ネットワーク機器・Webアプリ・産業制御システム（ICS）にわたる広範な製品が対象です。FCEB 機関は 4月〜5月中の修正期限が設けられており、KEV カタログに掲載済みの項目は期限内対応が義務付けられています。詳細は CISA の公式 NVD サマリーページで確認してください。
