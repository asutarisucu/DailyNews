shopify
🔗 Shopify Developer Changelog

Shopify Scripts が6月30日に完全廃止 — 残り31日、Shopify Functions への移行が急務

shopify
🔗 Shopify Developer Changelog

Shopify Scripts は2026年6月30日をもって完全廃止となる。残り31日で、ディスカウント・配送・決済ロジックを Scripts で実装しているストアは当日から動作が停止する。公式の移行先は WebAssembly ベースの Shopify Functions。4月15日以降は新規 Scripts の編集・公開も不可となっており、移行が未完了の開発者は Shopify admin の「Scripts カスタマイズレポート」から代替手段を確認することが推奨されている。

🔗 Anthropic

Anthropic は Google および Broadcom と、次世代 TPU 容量を複数ギガワット規模で確保する多年契約を締結した。合計約 3.5GW のコンピュートを 2027 年以降に順次オンライン化する計画で、NVIDIA への依存を分散させる狙いもある。背景には Anthropic の年間収益（ランレート）が $300億を超え、$100万以上を支出するビジネス顧客が 1,000 社を超えるなど急成長がある。インフラの大半は米国内に設置予定。

ChatGPhish — ChatGPT の Markdown 暗黙的信頼を悪用したプロンプトインジェクション攻撃が公開、フィッシングサイトへの誘導や QR コード表示が可能に

ai
🔗 The Hacker News

Permiso Security の研究者が「ChatGPhish」と命名した ChatGPT の脆弱性を公開した（5月29日）。chatgpt.com のレスポンスレンダラーが、要約した第三者ページ由来の Markdown リンクと画像 URL を暗黙的に信頼するため、攻撃者が細工した Web ページを ChatGPT に要約させるだけで、信頼された AI インターフェース内にフィッシング URL・偽セキュリティアラート・インライン QR コードが表示される。研究者は4月29日に Bugcrowd 経由で OpenAI へ報告済みだが、公開時点では修正確認が取れていない。

🔒 セキュリティ

Ghost CMS CVE-2026-26980（CVSS 9.4）が 700 以上のサイトで悪用 — SQL インジェクションで Admin API キーを窃取し ClickFix 攻撃に悪用

security
🔗 The Hacker News

Ghost CMS の Content API に存在するブラインド SQL インジェクション脆弱性 CVE-2026-26980（CVSS 9.4）が、ハーバード大学を含む 700 以上のサイトで悪用されていることが判明した。攻撃者は filter=slug:[...] クエリパラメータに細工したペイロードを挿入し、Admin API キーをはじめとした認証情報を認証なしで窃取。その後 Ghost Admin API でサイトの記事を改ざんし、悪意ある JavaScript ローダーを注入して偽 CAPTCHA（ClickFix）攻撃に悪用している。バージョン 6.19.1 で修正済み。バージョン 3.24.0〜6.19.0 のユーザーは即時アップグレードと全クレデンシャルのローテーションが必要。

PAN-OS CVE-2026-0300（CVSS 9.3）— Palo Alto ファイアウォールの認証不要 RCE が積極悪用中、ルート権限での任意コード実行が可能

security
🔗 The Hacker News

Palo Alto Networks の PAN-OS に存在するバッファオーバーフロー脆弱性 CVE-2026-0300 が野生で積極的に悪用されている。User-ID 認証ポータルサービスへ特殊なパケットを送信することで、未認証の攻撃者がルート権限でリモートコード実行が可能。PA-Series および VM-Series ファイアウォールが対象で、Cloud NGFW や Panorama には影響なし。攻撃者は nginx ワーカープロセスにシェルコードをインジェクションし、スパイ活動に利用している事例が報告されている。緩和策として User-ID 認証ポータルへのアクセスを信頼済みゾーンのみに制限するか、不要な場合は無効化することが推奨される。

TanStack npm サプライチェーン攻撃 — 42 パッケージ・84 バージョンが汚染、CI 認証情報・AWS/GCP/SSH キーを窃取するマルウェアを配布

security
🔗 TanStack Blog

5月11日に @tanstack/* の 42 パッケージ・84 バージョンが汚染されたことが判明。攻撃者は pull_request_target「Pwn Request」パターンと GitHub Actions キャッシュポイズニングを組み合わせてパッケージを掌握し、AWS IMDS/Secrets Manager・GCP メタデータ・Kubernetes トークン・npm/GitHub/SSH クレデンシャルを外部サーバーへ送信するマルウェアを注入した。週間 1,270 万ダウンロードの @tanstack/react-router も対象。外部研究者が20〜26分で検知したが、5月11日にインストールした開発者はクレデンシャルの全ローテーションが必要。Mistral AI・UiPath を含む 160 以上の npm/PyPI パッケージにも波及した。

Vercel Sandbox のファイルシステム永続化が GA — セッション間でファイルシステム状態を自動保存・復元、スナップショット管理が不要に

cloud
🔗 Vercel Changelog

Vercel Sandbox の永続化機能が5月26日に一般提供（GA）となった。サンドボックスが停止するとファイルシステム状態が自動保存され、再開時に復元される。手動でスナップショットを作成・追跡する必要がなく、サンドボックスは固有の名前で識別・再開できる。デフォルトで永続化が有効になっており、一時的なワークロードには永続化を無効にしてストレージコストを最小化するオプションも用意されている。AI エージェントのコーディング環境ユースケースで特に有用。
