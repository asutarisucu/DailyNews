Web Bot Auth 義務化へ — Storefront API・オンラインストアページへのボット/AI エージェントに厳格なレート制限を適用、公開鍵署名で上限緩和

shopify
🔗 Shopify Developer Changelog

Shopify が Storefront API とホステッドオンラインストアページへのボット・AI エージェントアクセスに対して厳格なレート制限を適用開始した。署名なしのリクエストは最も制限の厳しいティアが適用され、上限緩和には Web Bot Auth による HTTP 署名ヘッダーが必要となる。Web Bot Auth は公開鍵・秘密鍵ペアを用いた署名プロトコルで、Cloudflare ガイドの手順に沿って実装する。AI 経由注文が前年比 11 倍に急増した Shopify が、急増するエージェントトラフィックへの対処として導入した措置。

shopify
🔗 Shopify Developer Changelog

GraphQL Admin API の Order オブジェクトに cartToken フィールドが追加され、注文の作成に使われたカートのトークンを取得できるようになった。これは REST Admin API の cart_token フィールドに相当する機能で、GraphQL への完全移行を進める開発者にとって長年の課題が解消された。カートと注文の紐付けを行うアナリティクスやアフターオーダーフローの実装が大幅に簡素化される。

🔗 Anthropic API Docs

Anthropic が Claude Developer Platform のキャッシュ診断機能をパブリックベータとして公開した。Messages リクエストに diagnostics.previous_message_id を付与し、cache-diagnosis-2026-04-07 ベータヘッダーを含めると、API がプロンプトキャッシュプレフィックスの不一致箇所と理由（cache_miss_reason）を自動で返す。これまで手動でデバッグする必要があったキャッシュヒット率の改善作業が大幅に短縮され、コスト最適化を図る開発者にとって実用性の高い機能となっている。

Next.js 5月セキュリティリリース — CVE-2026-27979（DoS/メモリ枯渇）と CVE-2026-29057（HTTP リクエストスマグリング）に対応、15.5.13/16.1.7/14.2.35 以上へ更新を

security
🔗 Vercel Changelog

Vercel が Next.js の 5 月セキュリティリリースを公開した。CVE-2026-27979 は App Router の Partial Prerendering 有効環境で next-resume ヘッダーを持つ過大な POST ペイロードによりメモリ枯渇が発生する DoS 脆弱性（16.0.1〜16.1.6 が対象）。CVE-2026-29057 は rewrites プロキシが使用する http-proxy ライブラリの欠陥で、Transfer-Encoding: chunked な DELETE/OPTIONS リクエストでリクエスト境界の不整合が発生し内部エンドポイントへのスマグリングが可能。Next.js 15.x は 15.5.13 以上、16.x は 16.1.7 以上、14.x は 14.2.35 以上へのアップデートが必須。
