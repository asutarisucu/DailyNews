GraphQL Admin API の Order オブジェクトに cartToken フィールドを追加 — REST と同等の注文・カート紐づけが GraphQL で可能に

shopify
🔗 shopify.dev/changelog

GraphQL Admin API の Order オブジェクトに cartToken フィールドが追加され、注文の元となったカートトークンに REST API と同様にアクセスできるようになりました。これまで GraphQL のみで注文とカートを紐づけるには REST API を併用する必要がありましたが、本更新により GraphQL 単体での実装が可能になります。カスタムチェックアウト・注文アトリビューション分析・カート復元ロジックを GraphQL API のみで完結させたいアプリ開発者にとって重要なアップデートです。

マーケット別ディスカウント設定が解禁 — 地域・B2B・店舗ロケーション単位で割引対象を絞り込み

shopify
🔗 Shopify Changelog

DiscountContextInput に markets オプションが追加され、地域（国・地域）・B2B 企業ロケーション・小売店舗ロケーション単位でディスカウントの適用対象を指定できるようになりました。Basic・BXGY・App・Free Shipping の全ディスカウントタイプ（自動・コードベース両方）に対応しています。なお eligibility は markets か customer segments のどちらか一方のみ設定可能で、API バージョン 2026-07 以降でのサポートとなります（旧バージョンでは市場対象ディスカウントはフィルタリング除外されます）。

Storefront API・ストアページへのボットアクセスに厳格なレート制限を適用 — 上限緩和は Web Bot Auth で対応

shopify
🔗 shopify.dev/changelog

Shopify が Storefront API および Shopify ホスト型オンラインストアページへのボット・エージェントアクセスに対し、より厳格なレート制限を適用すると発表しました。スクレイパーや自動クローラーが現行制限を超えてアクセスする場合はリクエストがブロックされます。より高いレート上限が必要なボット運営者は Web Bot Auth を利用して認証することで制限の緩和が可能です。SEO クローラー・価格比較ツール・在庫モニタリングなど Storefront API を活用した自動化ツールを運用している場合は対応が必要です。

security
