Shopify Scripts 廃止まで残り16日 — 6月30日に全スクリプト停止、Shopify Functions への移行が必須

shopify
🔗 Shopify Developer Changelog

6月30日をもって Shopify Scripts が完全廃止となり、カスタム Ruby スクリプトによるディスカウント・配送・決済カスタマイズロジックが停止する。移行先は WebAssembly ベースの Shopify Functions で、アーキテクチャが根本的に異なるため単純な書き換えでは対応できない。すでに新規スクリプトの編集・公開は4月15日に停止されており、今回の期限は最終（過去2度延長済み）。未対応のストアは「Shopify Scripts customizations report」でリスト化し、即時移行を開始することが強く推奨される。

Streamlined Metaobject API — values プロパティ追加でシリアライズ不要の一括フェッチ・更新が可能に（GraphQL 2026-07）

shopify
🔗 Shopify Developer Changelog

GraphQL Admin API バージョン 2026-07 にて、Metaobject に新たな values プロパティが追加された。単一コールで全フィールドを JSON 互換オブジェクトとして取得でき、アプリ側でのデシリアライズ処理が不要になる。create/update 時も JSON スタイルのオブジェクトを渡すだけで API 側がシリアライズを処理する。omit したフィールドはクリアされる挙動に注意が必要で、既存のフィールドを保持する場合は全フィールドを明示的に指定する必要がある。

ai
🔗 Anthropic News

OpenSSL CVE-2026-45447 パッチリリース — AI との協働で発見した PKCS#7 ヒープ解放後使用バグ、RCE リスクあり

security
🔗 SecurityWeek

OpenSSL が CVE-2026-45447（高深刻度）を修正するパッチをリリースした。PKCS#7 / S/MIME 署名メッセージを処理する際、digestAlgorithms フィールドが空の ASN.1 SET の場合に呼び出し元所有の BIO が誤って解放され、ヒープ破壊・クラッシュ・場合によっては RCE が発生する可能性がある。本脆弱性は Claude AI との協働研究によって発見された点でも注目を集めている。対応バージョンは OpenSSL 4.0.1・3.6.3・3.5.7・3.4.6・3.0.21（後2者は各プレミアムサポート顧客向けに 1.1.1zh・1.0.2zq も）。未パッチ環境では不審なソースからの PKCS#7 / S/MIME メッセージ処理を回避することを推奨。

security
