Shopify CLI の app release --force フラグが非推奨に — 今後のリリースで削除予定、App Automation Tokens への移行を推奨

shopify
🔗 Shopify Developer Changelog

Shopify CLI の shopify app release コマンドで使用できる --force フラグが非推奨（deprecated）となり、今後のリリースで削除される予定が発表された。--force フラグは CI/CD パイプライン内で確認プロンプトをスキップするために使われてきたが、アプリリリースの安全性を高めるため廃止される。今後は App Automation Tokens を利用したCI/CDデプロイが推奨される代替手段となる。既存パイプラインに --force フラグを含んでいる場合は、削除前に移行作業を実施することが強く推奨される。

🔗 CISA

🔗 Microsoft Docs

Microsoft Defender for Cloud が Microsoft Defender ポータルに統合され、Azure・AWS・GCP にまたがるハイブリッドおよびマルチクラウド環境のセキュリティ脅威を一元的に管理・対応できるようになった。これまで別々のポータルで管理が必要だった各クラウドの脅威検出・インシデント対応・セキュリティ推奨が単一のビューに統合され、SOC チームの運用効率が大幅に向上する。XDR（拡張検知・応答）機能との連携も強化され、エンドポイント・ID・メール・クラウドワークロードにまたがる脅威相関分析が可能になる。Azure を中心にマルチクラウドを運用している組織は Defender ポータルへの移行によりセキュリティ運用の統合と簡素化を進められる。

Vercel Sandboxes がファイルシステム永続化ベータを公開 — セッション間のFS状態を自動保存・復元、デフォルトで永続化有効

dev
🔗 Vercel Changelog

Vercel Sandboxes がセッション間でのファイルシステム状態を自動的に保存・復元する永続化機能をベータとして公開した。永続化はデフォルトで有効になっており、スナップショットの手動管理なしにサンドボックスの実行状態を維持できる。サンドボックスはカスタマイズ可能な名前で一意に識別され、停止中のサンドボックスに対して runCommand() や writeFiles() を呼び出すと最新スナップショットから自動的にセッションが再開する。一時的なワークロード向けには --non-persistent フラグでストレージコストを削減でき、AI エージェントが状態を保持しながら非同期で長時間タスクを実行するユースケースに最適。

dev
🔗 GitHub Blog

GitHub Copilot が2026年6月1日よりプレミアムリクエスト数制限から AI クレジット制度（usage-based billing）へ移行する。Copilot Pro（$10/月）・Pro+（$39/月）のプラン料金は変わらず、月額相当の AI クレジットが毎月付与される（1 AI credit = $0.01 USD）。使用するモデルとトークン消費量に応じてクレジットが消費され、不足時は追加購入が可能。月額プランは6月1日に自動移行されるが、年間プランは更新まで既存の premium request ベース課金が継続される。Business・Enterprise はクレジットが組織全体でプール管理される形になり、移行プロモーションとして7〜8月は追加 included usage が付与される。

other
🔗 SecurityWeek
