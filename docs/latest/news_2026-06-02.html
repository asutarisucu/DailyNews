【残28日】Shopify Scripts が6月30日に廃止 — Shopify Functions への移行期限まで約4週間、既存スクリプトは同日に停止

shopify
🔗 Shopify Developer Changelog

Shopify Scripts は2026年6月30日をもって全面廃止となり、公開済みのスクリプトもこの日を境にすべて実行が停止される。4月15日以降は新規スクリプトの作成・編集・公開がすでに不可能になっており、現在実行中のスクリプトも猶予期間（残28日）に移行を完了させなければチェックアウト・送料・支払いカスタマイズが機能しなくなる。移行先は Shopify Functions（Rust / JavaScript / AssemblyScript でコンパイル後 WASM として実行）で、サブ5ms の安定した実行と冷スタートなしを実現。移行手順は公式ドキュメントに詳細があり、商用機能への影響を避けるため6月15日を目安に対応することを推奨する。

製品バリアントを製品とは独立して公開/非公開に — API 2026-07 で Publishable インターフェースをバリアントに拡張、チャネル・カタログ単位の細粒度制御が可能に

shopify
🔗 Shopify Developer Changelog

API バージョン 2026-07 より、ProductVariant が Publishable インターフェースを実装し、製品を削除・複製・ストアフロントコードで非表示にすることなく、チャネルやカタログ単位でバリアントを個別に公開/非公開に切り替えられるようになった。バリアントはデフォルトで公開状態（オプトアウト型）で、製品レベルの公開設定が引き続き優先されるため既存アプリへの破壊的変更はない。事前公開したくないバリアントを variant.published: false で作成できるほか、特定チャネルへの露出を細かくコントロールできる。B2B カタログや限定販売戦略の実装が大幅に簡素化される。

【本日期限】Microsoft Defender 脆弱性 CVE-2026-41091/CVE-2026-45498 が野生で悪用中 — CISA KEV に追加、連邦機関は6月3日までにパッチ適用必須

security
🔗 Help Net Security

Microsoft Defender（Malware Protection Engine）に2件の脆弱性が発見されており、両方とも野生での積極的な悪用が確認されている。CVE-2026-41091（CVSS 7.8）は Malware Protection Engine がリンクを解決する際の処理ミスを悪用した特権昇格脆弱性で、SYSTEM 権限が取得可能。CVE-2026-45498（CVSS 4.0）は Defender をサービス不能（DoS）状態にする脆弱性。CISA が両件を KEV カタログに追加し、連邦民間行政機関（FCEB）への修正期限は本日6月3日。修正済み Malware Protection Engine v1.1.26040.8 は Windows Update 経由で自動配布されているが、エアギャップ環境や更新が遅延している場合は手動適用が必要。

CVE-2026-31431「Copy Fail」— Linux kernel の特権昇格脆弱性が CISA KEV に追加、2017年以降のほぼ全ディストリビューションに影響

security
🔗 The Hacker News

Linux カーネルの暗号サブシステムに存在するローカル特権昇格脆弱性 CVE-2026-31431（CVSS 7.8、通称「Copy Fail」）が CISA の Known Exploited Vulnerabilities カタログに追加された。未特権のローカルユーザーが 732 バイトの Python ベースのエクスプロイトでカーネルのページキャッシュを破壊し、ほぼ100%の成功率でルート権限を取得できる。Ubuntu・RHEL・Amazon Linux を含む2017年以降のほぼ全メジャーディストリビューションが影響を受ける。Go・Rust 版の実装も OSS リポジトリで確認されており、攻撃の高度化が懸念される。Linux カーネル 6.18.22・6.19.12・7.0 以降で修正済み。

security

dev
🔗 TypeScript Blog
