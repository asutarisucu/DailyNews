製品バリアントを製品とは独立して公開/非公開に — API 2026-07 で Publishable インターフェースをバリアントに拡張、チャネル・カタログ単位の細粒度制御が可能に

shopify
🔗 Shopify Developer Changelog

API バージョン 2026-07 より、ProductVariant が Publishable インターフェースを実装し、製品を削除・複製・ストアフロントコードで非表示にすることなく、チャネルやカタログ単位でバリアントを個別に公開/非公開に切り替えられるようになった。バリアントはデフォルトで公開状態（オプトアウト型）で、製品レベルの公開設定が引き続き優先されるため既存アプリへの破壊的変更はない。事前公開したくないバリアントを variant.published: false で作成できるほか、特定チャネルへの露出を細かくコントロールできる。B2B カタログや限定販売戦略の実装が大幅に簡素化される。

全パブリックアプリへの Expiring Offline Access Token が2027年1月1日に義務化 — 4月1日からの新規アプリ要件を既存アプリ全体に拡大

shopify
🔗 Shopify Developer Changelog

2026年4月1日以降に作成された新規パブリックアプリに適用されていた Expiring Offline Access Token（有効期限付きオフラインアクセストークン）の要件が、2027年1月1日より既存の全パブリックアプリに拡大されることが正式に発表された。セキュリティ強化を目的とした措置で、Admin API への呼び出しに旧来の無期限トークンを使い続けているアプリは、期限までにトークン更新ロジックの実装が必要。Shopify は対応手順とガイドを開発者ドキュメントで提供しており、猶予期間は残り約7か月。

SubscriptionBillingAttempt の errorCode・errorMessage フィールドが API 2026-04 から非表示に — processingError.code / processingError.message への移行が必要

shopify
🔗 Shopify Developer Changelog

SubscriptionBillingAttempt オブジェクトの errorCode および errorMessage フィールドが、API バージョン 2026-04 から非表示（hidden）となった。両フィールドは 2025-01 で processingError が導入された際に非推奨（deprecated）となっており、今回の変更で実質的に削除された形になる。定期課金アプリはクエリを processingError.code および processingError.message を使用するよう更新する必要がある。2026-04 以降のバージョンで旧フィールドを参照しているアプリはエラーハンドリングが機能しなくなる。

【明日期限】Microsoft Defender 脆弱性 CVE-2026-41091/CVE-2026-45498 が野生で悪用中 — CISA KEV に追加、連邦機関は6月3日までにパッチ適用必須

security
🔗 Help Net Security

Microsoft Defender（Malware Protection Engine）に2件の脆弱性が発見されており、両方とも野生での積極的な悪用が確認されている。CVE-2026-41091（CVSS 7.8）は Malware Protection Engine がリンクを解決する際の処理ミスを悪用した特権昇格脆弱性で、SYSTEM 権限が取得可能。CVE-2026-45498（CVSS 4.0）は Defender をサービス不能（DoS）状態にする脆弱性。CISA が両件を KEV カタログに追加し、連邦民間行政機関（FCEB）への修正期限は明日6月3日。修正済み Malware Protection Engine v1.1.26040.8 は Windows Update 経由で自動配布されているが、エアギャップ環境や更新が遅延している場合は手動適用が必要。

CVE-2026-31431「Copy Fail」— Linux kernel の特権昇格脆弱性が CISA KEV に追加、2017年以降のほぼ全ディストリビューションに影響

security
🔗 The Hacker News

Linux カーネルの暗号サブシステムに存在するローカル特権昇格脆弱性 CVE-2026-31431（CVSS 7.8、通称「Copy Fail」）が CISA の Known Exploited Vulnerabilities カタログに追加された。未特権のローカルユーザーが 732 バイトの Python ベースのエクスプロイトでカーネルのページキャッシュを破壊し、ほぼ100%の成功率でルート権限を取得できる。Ubuntu・RHEL・Amazon Linux を含む2017年以降のほぼ全メジャーディストリビューションが影響を受ける。Go・Rust 版の実装も OSS リポジトリで確認されており、攻撃の高度化が懸念される。Linux カーネル 6.18.22・6.19.12・7.0 以降で修正済み。

security
🔗 The Hacker News

dev
🔗 TypeScript Blog

TypeScript 6.0 の安定版がリリースされた。TypeScript コンパイラが JavaScript で実装された最後のメジャーバージョンで、次の TypeScript 7 は Go 言語での全面書き直しとなる。主な破壊的変更として、ES5 ターゲット・AMD・UMD モジュールシステム・baseUrl 設定・outFile バンドリングが廃止された。また import defer のサポートが追加されたほか、型検査のパフォーマンスも改善。TypeScript 7 (Go 版) では並列処理と高速なネイティブバイナリによりコンパイル速度の大幅向上が見込まれており、TypeScript 6.0 への移行は TS7 に向けた準備段階として位置付けられる。
