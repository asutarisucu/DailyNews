Automatic Discount Functions が B2B セッションに対応 — オンラインストアと同一の割引ロジックを企業間取引にも適用可能に

shopify
🔗 Shopify Developer Changelog

GraphQL Admin API にて、Shopify Functions で実装した自動割引（Automatic Discount）が B2B セッションにも適用されるようになった。これまで B2B の購入セッションでは自動割引が機能せず、B2C 専用として扱われていたため、企業向け価格設定には別途の実装が必要だった。今回の対応により、既存の割引ロジックをそのまま B2B チャネルに展開でき、オムニチャネルの一貫した価格体験を実現できる。新たな Function の実装は不要で、既存の Automatic Discount がそのまま有効となる。

Discount Functions が Shopify POS に対応 — オンラインと実店舗で割引ロジックを一元管理するオムニチャネルが実現

shopify
🔗 Shopify Developer Changelog

Shopify Point of Sale（POS）において Discount Functions が正式サポートされ、オンラインストアと同一の割引ロジックを実店舗のレジ端末でも利用できるようになった。これまで POS での割引は別途のスクリプトやマニュアル割引に依存しており、オンラインとオフラインで体験の差が生じていた。今回の対応により Shopify Functions で一元管理した割引ロジックが全チャネルで一貫して機能し、真のオムニチャネル割引体験が実現する。Shopify Scripts から Shopify Functions への移行が済んでいれば、追加実装なしで POS にも適用される。

Microsoft Patch Tuesday 6月 — 206件の脆弱性に対応、CVSS 9.8 のゼロデイ2件（CVE-2026-45657・CVE-2026-47291）含む Critical 37件

security
🔗 Zero Day Initiative

Microsoft の2026年6月パッチチューズデーで206件の脆弱性が修正され、うち37件が Critical、3件が公開済みのゼロデイだった。特に CVE-2026-45657（CVSS 9.8）と CVE-2026-47291（CVSS 9.8）はいずれも認証不要のリモートコード実行を許す高深刻度で、前者はシステム権限での任意コード実行、後者はユーザー操作なしで悪用可能な点が際立つ。Windows・Office・Azure の広範なコンポーネントが対象で、エンタープライズ環境では月例の更新適用を優先的に進めることが強く推奨される。

Chrome 149 V8 ゼロデイ CVE-2026-11645 が緊急パッチ — CISA が KEV カタログに追加、細工HTMLでの任意コード実行を許す高リスク脆弱性

security
🔗 CISA Known Exploited Vulnerabilities

Google は Chrome 149 に対して緊急アップデートをリリースし、V8 JavaScript エンジンに存在するゼロデイ脆弱性 CVE-2026-11645 を修正した。攻撃者は細工した HTML ページを通じて任意のコードを実行できるため、実際の悪用が確認された段階で緊急対応となった。CISA は本脆弱性を Known Exploited Vulnerabilities（KEV）カタログに追加し、連邦機関に対して期限内のパッチ適用を義務付けた。Chrome の設定メニューから「Chrome について」を開くと自動更新が適用されるため、全ユーザーに即時確認が推奨される。

Palo Alto PAN-OS CVE-2026-0257 が野生で悪用中 — GlobalProtect ポータルの認証バイパスで攻撃者が内部ネットワークに不正侵入

security
🔗 The Hacker News

Palo Alto Networks の PAN-OS に存在する認証バイパス脆弱性 CVE-2026-0257 が GlobalProtect ポータル・ゲートウェイを標的にした実際の攻撃で悪用されていることが確認された。認証オーバーライドクッキーの検証不備に起因し、未認証のリモート攻撃者が正規の VPN 接続を確立して内部ネットワークへのアクセスを獲得できる。CISA が KEV カタログに追加済みで、Rapid7 は複数顧客環境での第二波攻撃も確認している。緩和策として認証オーバーライド機能の無効化、または認証オーバーライド専用の新規証明書への切り替えが推奨される。

🛠️ 開発ツール・OSS

npm v12 の破壊的セキュリティ変更を事前告知 — allowScripts・--allow-git・--allow-remote がデフォルト無効化、7月リリース予定

dev
🔗 GitHub Changelog

2026年7月リリース予定の npm v12 で予告された破壊的変更が注目を集めている。allowScripts がデフォルトでオフになることでインストール時の preinstall/install/postinstall スクリプト自動実行が禁止され、ネイティブアドオン（node-gyp）も影響を受ける。あわせて --allow-git と --allow-remote もデフォルト無効化され、Git リポジトリや外部 URL 依存のインストールには明示的な許可が必要になる。これらはサプライチェーン攻撃リスク低減を目的とした変更で、npm 11.16.0 以降で警告として事前確認が可能なため、影響を受けるプロジェクトの早期棚卸しが推奨される。

Dependabot が Deno エコシステムのバージョン更新に対応 — deno.json の依存関係を自動 PR で管理可能に

dev
🔗 GitHub Changelog

GitHub の Dependabot が Deno エコシステムのバージョン更新に対応し、.github/dependabot.yml に Deno エントリを追加するだけで deno.json の依存関係に対する自動 PR 作成が可能になった。Deno を採用するプロジェクトで依存関係のアップデート管理が大幅に改善され、手動での更新作業が不要になる。現時点ではバージョン更新のみの対応でセキュリティアップデートには未対応だが、Deno エコシステムの成熟に合わせて機能拡張が期待される。Node.js・Python に続く GitHub の Deno サポート強化の一環。
