Shopify Flow アクション拡張が相対パスの URL をサポート — dev トンネル切り替え時の runtime_url 更新が不要に

shopify
🔗 Shopify Developer Changelog

Flow アクション拡張の設定プロパティ（runtime_url・validation_url・config_page_url・config_page_preview_url）で、絶対 HTTPS URL に加えてスラッシュ始まりの相対パス（例: /api/flow/actions/place-bid）が使えるようになった。相対パスを使うと Shopify CLI は shopify app dev 実行中はトンネル URL に、デプロイ後はアプリの application_url に自動で解決するため、開発中にトンネル URL が変わるたびに runtime_url を書き直す手間が完全になくなる。ローカル開発の反復速度が向上し、CI/CD パイプラインに設定値をハードコードするリスクも排除できる。既存の絶対 URL 設定はそのまま動作するため、移行は任意のタイミングで段階的に進められる。

Shopify ディスカウント UI 拡張に purchaseType・recurringCycleLimit が追加 — サブスクリプション割引の細かな制御が UI から直接設定可能に

shopify
🔗 Shopify Developer Changelog

アプリのディスカウント UI 拡張の discounts プラグインで purchaseType と recurringCycleLimit の 2 フィールドが利用可能になり、従来は GraphQL Admin API 経由でしか設定できなかったサブスクリプション割引の挙動を拡張 UI から直接コントロールできるようになった。purchaseType は割引を一回購入・サブスクリプション・両方のいずれに適用するかを指定し、recurringCycleLimit はサブスクリプション割引を何回の請求サイクルに適用するかを数値で指定（0 = 無制限）。既存のすべてのアプリ割引は一回購入・サブスクリプション両方に適用済みとしてバックフィルされており、動作変更なし。新フィールドを利用するには拡張コードの discounts プラグイン設定を更新するだけでよい。

Shopify GraphQL Admin API 2026-07: アプリ所有の配送プロファイルに coversAllItems フィールド追加 — 全商品対応プロファイルかどうかを一発で判定可能に

shopify
🔗 Shopify Developer Changelog

GraphQL Admin API 2026-07 でアプリ所有の配送（Delivery）プロファイルに coversAllItems ブール値フィールドが追加された。このフィールドが true の場合、そのプロファイルがストア内のすべての出荷可能な商品バリアントに適用されることを示し、各商品や商品バリアントを明示的にプロファイルへ割り当てなくても全商品対応プロファイルであることを即座に判断できる。これまでは全商品対応かどうかを判定するために全バリアントの割り当てを確認する追加クエリが必要だったが、coversAllItems 一つで完結するようになりコードの複雑さが大幅に低減する。配送プロファイルを管理するアプリは 2026-07 移行時に当フィールドの活用を検討することが推奨される。

Deno 2.6 リリース: dx コマンドで npx 相当の体験を実現 + deno audit で依存関係の CVE チェック + tsgo で型チェック 2倍速化

dev
🔗 Deno Blog

Deno 2.6 がリリースされ、開発者体験を大幅に強化する複数の新機能が追加された。最大の目玉は dx コマンドで、npm および JSR パッケージのバイナリを npx と同じ手軽さで実行できる（デフォルトは --allow-all だが任意のパーミッションフラグを渡すことも可能）。deno audit サブコマンドにより GitHub CVE データベースを参照して JSR・npm 両方の依存関係の脆弱性レポートを生成できるようになり、サプライチェーンセキュリティの強化に直結する。型チェックには Go 製の実験的 TypeScript チェッカー tsgo を統合し、従来比最大2倍の高速化を達成。--ignore-read・--ignore-env による細粒度パーミッション、ソースフェーズインポート、ネイティブソースマップも追加されている。

dev
🔗 Deno Docs — Migration Guide
